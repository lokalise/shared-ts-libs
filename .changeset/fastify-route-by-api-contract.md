Add `buildFastifyApiRoute` for `defineApiContract` contracts, with support for SSE responses in addition to plain JSON routes. The handler shape is inferred from the contract's `responsesByStatusCode`: non-SSE contracts take a `(request, reply) => { status, body }` handler, while any contract that declares an SSE response (SSE-only or mixed with non-SSE via `anyOfResponses`) takes a single `(request, reply, sse) => { status, body }` handler. Every response is returned as `{ status, body }`; for an SSE status the `body` is an `AsyncIterable` of the contract's events (e.g. from an `async function*`), which the framework validates against the event schemas, streams, and closes. A mixed status's `body` is a union of the JSON payload and the event iterable, so one handler can serve both — the client preference can be resolved from the `Accept` header via the exported `determineMode` helper. For advanced streaming (keep-alive, lifecycle hooks, reconnection) the `sse` context's `start(...)` can drive the session imperatively instead. The route method, URL, request schemas and response schema are derived from the contract; SSE lifecycle hooks (`onConnect`, `onClose`, `onReconnect`, `serializer`, `heartbeatInterval`) are configurable via the options argument.

SSE-capable routes require the `@fastify/sse` plugin, which is now a peer dependency.
